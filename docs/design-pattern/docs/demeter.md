# 迪米特法则

Law of Demeter（LoD）也被称为最少知识原则（Least Knowledge Principle，LKP）。

::: tip
一个对象应该对其他对象有最少的了解。
:::

## 只和朋友交流

出现在成员变量、方法的输入输出参数中的类称为成员朋友类，而出现在方法体内部的类不属于朋友类。

## 朋友间是有距离的

朋友类中的 public 方法不应在其他类中被多次调用，应该修改方法权限且将方法的组合封装为一个新方法。

## 是自己的就是自己的

如果一个方法放在本类中，既不增加类间关系，也对本类不产生负面影响，那就放置在本类中。
